Ultimate Colorizer

The complete recolorization framework for Unity — textures, materials, meshes, sprites, tilemaps and palettes, plus runtime character dyes and cosmetics. Click-to-recolor in the Editor with no code, or drive everything from a clean C# API.

Unity 6000.0+ No-Code Ready Built-in & URP Editor + Runtime Source-available C#
ⓘ
One package, two ways to work. No-code: add a component, pick a color, and press Play — the object recolors with zero scripting. Code-first: the same components and a documented API give power users full control. The no-code layer is purely additive: it never overrides values you set from code.
✅
Non-destructive by default. Material recoloring uses MaterialPropertyBlock — your shared materials and source textures are never mutated. Preview in the Editor, undo freely.

⚡ Quick Start (No-Code)

Recolor a real object in the scene without writing a line of code.

Create a recolor object
GameObject → Ultimate Colorizer → Material Tint Cube. A cube drops in with a Recolor Material Applicator already attached.
Pick a color
In the Inspector, change Color. The cube updates live in the Scene view via the Preview In Scene button.
Make it run on its own
Tick Apply On Start. Now the color applies automatically when the game runs.
Press Play
The object recolors — no script. Try a Sprite Tint Object or a Runtime Dye Controller next.
💡
Preview vs. Apply On Start. Preview In Scene shows the result in the Editor right now. Apply On Start makes it happen automatically at runtime. Leave Apply On Start off if you'd rather drive recoloring from code.

🎨 Palettes & Import

Palette assets hold named, role-tagged colors and import/export as JSON.

Create a palette

Assets → Create → Ultimate Colorizer → Palette. The palette inspector shows a swatch strip, a one-line summary, and Import JSON / Export JSON buttons.

💡
Robust import. Importing a malformed or empty file fails gracefully with a clear dialog — it never throws or half-applies. The JSON format matches PaletteJsonUtility, which the runtime API uses too, so palettes round-trip between projects and tools.

Harmony generation

Generate palettes from a base color with PaletteGenerator.Generate(seed, mode, count). Modes include Complementary, Analogous, Triadic, Monochrome, Value Ramp, Split-Complementary, Tetradic, Saturation Ramp, Warm and Cool. The Palette Browser exposes this visually.

⚙ How It Works

A profile describes what to recolor; processors do the work; a service routes them.

RecolorProfile → RecolorTarget → Operations → IRecolorProcessor → RecolorResult
  1. A RecolorProfile (ScriptableObject) holds targets, palettes, regions, operations and runtime channels.
  2. Each RecolorTarget names a source (texture, material, mesh, sprite, tilemap, renderer, character) and a RecolorTargetType.
  3. An IRecolorProcessor self-selects per target (texture/material/mesh/tilemap) and applies the profile's operations.
  4. RecolorProcessingService routes a target to the right processor through a RecolorProcessorRegistry and returns a RecolorResult with generated objects and validation issues.
ⓘ
Operations are data. RecolorOperationType covers Replace Color, Replace Palette Entry, Hue Shift, Saturation/Value, Gradient Map, Tint, Multiply, Emission Color, Material Property Set, Vertex Color Replace and LUT Map — so a single profile drives many effects.

🖼 Textures & Sprites

CPU texture recoloring with deterministic, non-destructive output.

  • Replace color with tolerance and a choice of color-distance metric (RGB, weighted RGB, CIE76 approximation).
  • Palette remap from a source palette to a target palette.
  • Masks — threshold, channel, UV-rect, dilate/erode, combine.
  • Gradient map and 1D LUT mapping.
  • Sprite-rect recolor and contact-sheet generation for sprite sheets.
  • Smart color analysis clusters and classifies a texture's colors (skin, cloth, metal, emission…).
// Replace a color in a texture (preview copy; source is never mutated)
Texture2D result = TextureRecolorUtility.ReplaceColor(
    source, fromColor, toColor, tolerance: 24, preserveAlpha: true);C#

💎 Materials

Shader-aware material recoloring and variant generation.

  • Property-block recoloring — recolor a renderer without touching shared materials (see RecolorMaterialApplicator).
  • Shader analysis detects the pipeline (Built-in / URP / HDRP / Shader Graph) and the available color/texture properties.
  • Emission recoloring recognizes both _EmissionColor (Built-in/URP) and _EmissiveColor (HDRP/Lit).
  • Material variants — spin off recolored material copies; Material Preset Libraries store reusable color presets.
ⓘ
Base color resolves automatically. The engine prefers _BaseColor (URP/Lit), falls back to _Color (Built-in/Standard), then the first detected color property.

📐 Meshes & Tilemaps

Vertex colors, UV/submesh workflows, and non-destructive tilemap recoloring.

Meshes

  • Vertex color replacement with tolerance.
  • UV islands and UV-rect masks, submesh descriptors and channel maps.
  • Renderer hierarchy scanning to find and bind every renderer under a root.

Tilemaps

Recolor a UnityEngine.Tilemaps.Tilemap non-destructively: per-cell color replace, palette swap, HSV shift, multiply, tint, global tint, and color extraction. Per-cell colors live on the tilemap, not the tile, so a single shared tile recolors per cell.

🎭 Character Dye & Cosmetics

Player-facing customization: dye channels, cosmetic slots, deterministic randomize.

Dye channels

A RecolorProfile's Runtime Channels define named, palette-constrained dye channels (skin, hair, cloth, armor, weapon…) with default colors. A RecolorRuntimeController maps each channel to one or more renderer bindings and applies a RecolorRuntimeState at runtime.

// No-code: tick "Apply Defaults On Start" on the controller, or from code:
controller.SetChannelColor("cloth_primary", Color.cyan);
controller.ApplyProfileDefaults();   // build a state from the profile's defaultsC#

Cosmetics

CosmeticSlotApplicator toggles GameObjects per slot (helmet on/off, cape variants). CharacterRandomizer produces a deterministic dye state from a seed and a CharacterRandomizationRules asset (with locked channels and palette constraints).

💾 Save / Load

Serialize a player's appearance as lightweight data — not full textures.

  • JSON payloads via RuntimeCustomizationStore / RecolorRuntimeState.ToJson().
  • Compact, multiplayer-safe payloads via CompactRuntimePayloadCodec — small enough to sync over the network.
  • PlayerPrefs persistence helpers, plus a checksum to detect tampering/corruption.
💡
Robust by design. Loading an invalid payload is rejected gracefully (the API returns a failure and a message) rather than throwing.

⚒ Batch & SDK

Generate variants in bulk and extend the framework.

  • Batch processing with deterministic naming, dry-run, progress, cancellation and JSON/CSV reports (RecolorBatchProcessor).
  • Validation reports flag missing scripts, broken sample references, duplicate channels and release issues.
  • SDK — register custom shader profiles, processors, palette generators and validators; an extension registry discovers them by attribute.
  • Headless runners — validation and build entry points for CI (-executeMethod).

📙 Scripting API

The no-code components are thin wrappers over a clean, fully scriptable API.

using UltimateColorizer;

// 1) Recolor a renderer (non-destructive property block)
var applicator = go.AddComponent<RecolorMaterialApplicator>();
applicator.ApplyColor(Color.red);

// 2) Drive a recolor profile through the processing service
var registry = new RecolorProcessorRegistry();
registry.Register(new TextureRecolorProcessor());
var service = new RecolorProcessingService(registry);
RecolorResult result = service.Process(new RecolorContext(profile, target));

// 3) Runtime multi-channel dye
var state = new RecolorRuntimeState();
state.SetColor("skin", skinColor);
controller.Apply(state);
string json = controller.ToJson();   // save
controller.FromJson(json);          // loadC#
ⓘ
Code-first is never overridden. The no-code Apply On Start / Apply Defaults On Start toggles default to off, and the controller's auto-apply bails if a state was already applied from code. Existing scripted setups keep working unchanged.

💡 Troubleshooting

Common issues and quick fixes.

I added a component but nothing recolors on Play

Tick Apply On Start (it defaults to off so code-driven users aren't overridden). For RecolorRuntimeController, tick Apply Defaults On Start and make sure the profile has Runtime Channels and Renderer Bindings.

The color isn't changing / wrong shader property

Use the Color Property dropdown to pick the property your shader actually exposes — _BaseColor for URP/Lit, _Color for Built-in/Standard. The dropdown is populated from the assigned renderer's material.

Emission recolor does nothing in HDRP

HDRP/Lit uses _EmissiveColor (not _EmissionColor). The engine handles both, but verify your material has an emission property and that emission is enabled on the material.

Palette JSON import shows an error dialog

The file isn't valid palette JSON (or is empty). Export a palette first to see the expected format. Import fails gracefully and leaves the palette unchanged.

❓ FAQ

Quick answers.

Do I need to write code?

No. Add a component, pick a color, tick Apply On Start. The C# API is there when you want it.

Will it mutate my materials or textures?

No. Material recoloring uses property blocks; texture operations produce new copies. Your source assets are untouched.

Which render pipelines are supported?

Built-in and URP are supported and verified. HDRP is supported through pipeline-aware adapters; verify on an HDRP project before relying on it.

Does it need the Input System package?

No. It works with both the Input System package and the legacy Input Manager.

Can I use it for runtime player customization / multiplayer?

Yes. Dye state serializes to JSON or a compact, multiplayer-safe payload you can sync and persist.
