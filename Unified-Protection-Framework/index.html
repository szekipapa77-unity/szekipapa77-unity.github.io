Unified Protection Framework

A defense-in-depth toolkit for Unity: managed-code obfuscation, authenticated encryption, secure saves, protected assets, runtime integrity, anti-tamper signals and offline licensing — configured from a single profile, driven from a no-code dashboard, and automatable headlessly for CI.

Unity 6000.0 (Unity 6)+ Mono + IL2CPP standalone No-code editor layer Defense-in-depth
ⓘ
One profile, many layers. A single UpfProtectionProfile asset drives every layer — obfuscation, encryption, asset protection, integrity, IL2CPP and licensing. Pick a one-click preset (Safe / Balanced / Aggressive), then build a protected player from a menu item, the Dashboard, or a headless batch command.
⚠
Honest by design. UPF raises the cost of casual inspection and tampering — it does not make a client build immune to reverse engineering. Client-side keys can be extracted and runtime-decrypted data can be captured from memory. See Security Model & Limits before you ship.

⤓ Installation

Import the package, open the Welcome window, create a profile.

Import from the Asset Store
In Window → Package Manager → My Assets, download and import UPF. It lands at Assets/UPF/.
The Welcome window opens automatically
On first import the Welcome to UPF window appears once. Re-open it any time from Window → UPF → Welcome or Tools → UPF → Welcome.
Create a Protection Profile
Tools → UPF → Create or Select Protection Profile creates Assets/UPFProfiles/DefaultUPFProfile.asset and selects it. Pick a preset in the Inspector.
Open the Dashboard
Tools → UPF → Dashboard — analyze your project, dry-run, validate and build, all from one window.

Requirements

  • Unity 6000.0 (Unity 6) or newer. Developed and validated against 6000.0.62f1.
  • Mono and IL2CPP standalone builds both receive the managed-assembly obfuscation pass.
  • Addressables (com.unity.addressables) only if you use the Addressables protection bridge — the base package does not require it.

⚡ Quick Start

From a fresh import to a protected build, the short path.

Open the Dashboard
Tools → UPF → Dashboard. Select or create a protection profile at the top.
Check the active target
The dashboard shows your active Build Target Group / Build Target and which platform override (if any) resolves for it.
Click Analyze
UPF scans for reflection, serialization, UnityEvents, scenes, prefabs, Addressables, PlayerPrefs and IL2CPP settings, and writes a report. Fix any strict findings and add preserve rules where flagged.
Click Dry Run
See exactly what obfuscation would rename and encrypt — without modifying a build. Review the report in the profile's report folder.
Click Build Protected Player
Obfuscation + a signed integrity manifest are applied per the active profile. Output goes to Builds/UPFProtected/<target>; the report goes to Artifacts/UPF.
(Optional) Add a runtime guardian
Drop the UPF Protection Guardian component into your scene for no-code runtime anti-tamper. See Editor Tools.

Headless equivalent

# Dry-run protection against a specific profile
Unity.exe -batchmode -quit -projectPath . \
  -executeMethod UPF.Editor.Batch.DryRun \
  -upfProfile Assets/UPFProfiles/DefaultUPFProfile.assetPowerShell
💡
Ship checklist. Before distribution, run UPF.Editor.Batch.ValidatePackage and UPF.Editor.Batch.FinalAudit, set a real asset-encryption secret (not the development password), and confirm reports show no strict blockers.

⚙ How It Works

Protection is configured once, applied at build time, and verified at runtime.

Configure profile → Analyze / Dry Run → Build Protected Player → Ship → Runtime guardian verifies
  1. Configure a UpfProtectionProfile: choose a preset, set encryption keys and integrity/licensing options. Platform overrides resolve automatically per Build Target.
  2. Analyze / Dry Run the project so you know what will be renamed/encrypted and which symbols need preserve rules — before you commit to a build.
  3. Build a protected player. For both Mono and IL2CPP the managed-assembly obfuscation pass runs against staged assemblies; a signed integrity manifest is generated and a public-key trust anchor is shipped in upf-runtime-protection.json.
  4. Ship the player. The private signing key is written only to the report/artifacts folder — never into the player.
  5. At runtime, UpfRuntimeProtectionBootstrap (or the no-code Protection Guardian) verifies the signed manifest and runs anti-tamper checks, then applies the configured UpfIntegrityResponse (e.g. fail-closed quit, warn, or a custom callback).
ⓘ
Under IL2CPP, obfuscation runs before Unity converts IL to C++. The UpfScriptDllPostProcessor (an IPostBuildPlayerScriptDLLs hook) obfuscates staged user assemblies in place, so renamed private members and constant transforms propagate into the native binary and into global-metadata.dat. See IL2CPP.

🔐 Code Obfuscation

Conservative managed-assembly rewriting that preserves Unity serialization and reflection.

UPF's UpfCecilManagedProtector performs a conservative rewrite of eligible user assemblies during a protected build. The current pass:

  • renames private, non–Unity-sensitive symbols;
  • encrypts string literals (a reversible anti-grep measure, decoded at runtime by UpfStringProtector);
  • encodes supported numeric constants;
  • optionally inserts conservative control-flow / dead-code noise in methods without exception handlers;
  • writes a name mapping to UPF_Mapping.csv for support diagnostics.

Presets and renaming modes

ModeWhat it doesStrict mode
SafeRenames private members only; preserves all Unity serialization / reflection. Safest default.Off
BalancedBroader renaming with string + constant encryption, still conservative around Unity. Recommended for most projects.Off
AggressiveMaximum renaming plus control-flow noise and strict-mode validation. Test thoroughly before shipping.On

Preserve & exclude rules

When the scanner flags a string-based reflection target, a serialized DTO, or a public API, add it to the profile:

  • obfuscation.preserveNamePatterns — names matching these wildcard patterns are never renamed (e.g. MyNamespace.*, *Settings, System.Void Namespace.Type::Method()). Supports * and ?.
  • obfuscation.excludeAssemblyPatterns — whole assemblies skipped entirely (e.g. Assembly-CSharp, ThirdParty.*).

Decoding obfuscated stack traces

UpfStackTraceMapper reads UPF_Mapping.csv and restores original symbol names in player logs for support. Run it headlessly:

Unity.exe -batchmode -quit -projectPath . \
  -executeMethod UPF.Editor.Batch.DecodeStackTrace \
  -upfMapping Artifacts/UPF/UPF_Mapping.csv \
  -upfStackTraceInput player.log -upfStackTraceOutput decoded.txtPowerShell
⚠
Keep UPF_Mapping.csv private. It maps obfuscated names back to your originals — never include it in a distributed package. Obfuscation is conservative around Unity messages, compiler-generated code and methods with exception handlers; it is not a guarantee against reverse engineering.

🔑 Encryption & Secure Save

Authenticated encryption containers: AES-256-CBC for confidentiality, HMAC-SHA-256 for integrity.

The core primitive is UpfCryptoService, which protects/unprotects byte arrays. Higher-level services build on it: secure saves, encrypted config TextAssets, and encrypted files / StreamingAssets. Wrong keys, wrong context, or tampered payloads fail authentication by design.

Secure saves

using UPF.Runtime;

var service = new UpfSecureSaveService(crypto);
service.Save("slot1", new SampleSave { level = 3, playerName = "Player" });

var loaded = service.Load<SampleSave>("slot1");
if (loaded.Success) {
    // use loaded.Value
}C#

Encrypted config / TextAssets

Encrypt a config file from the menu (Tools → UPF → Encrypt Asset File...) or headlessly, then load it at runtime:

// Encrypt a config TextAsset (batch)
Unity.exe -batchmode -quit -projectPath . \
  -executeMethod UPF.Editor.Batch.ProtectTextAsset \
  -upfInputTextAsset Assets/Game/Config/config.json \
  -upfOutputTextAsset Assets/Game/Config/protected-config.bytes \
  -upfTextAssetContext configPowerShell
// Load & decrypt at runtime
var loader = new UpfEncryptedConfigService(crypto, "config");
UpfResult<string> json = loader.LoadJson(protectedConfigTextAsset, "config-key");C#

Key provisioning

UpfProtectionProfile.CreateAssetCrypto(requireSecureKey) is the production factory for asset / config encryption. It resolves the key from out-of-repo sources in priority order:

PrioritySourceProfile field
1Environment variable (e.g. injected by CI)encryption.productionKeyEnvVar (default UPF_ASSET_KEY)
2External key fileencryption.productionKeyFilePath
3Development password (local only)encryption.developmentPassword

When requireSecureKey is true (release / strict builds), UPF throws rather than encrypt with the default placeholder password. CreateDevelopmentCrypto() remains for local work and transparently honours the same production sources when present.

Platform secure storage

UpfPlatformSecureStorageProvider backs key material with OS keystores where available — Android Keystore (UpfAndroidKeystoreStorage.java, AES/GCM), iOS/tvOS Keychain (UpfAppleKeychain.mm) — with an editor/unsupported fallback.

⚠
Client-side keys can be extracted by skilled attackers, and runtime-decrypted data can be captured from memory. Treat encryption as one defense layer, not absolute secrecy. Never store server secrets in a client build. Production key plugins must be validated on real target devices and OS versions.

📦 Asset / Addressables Protection

Encrypt StreamingAssets, AssetBundles and local Addressables bundles — decrypted transparently at load.

What you can protect

📁 StreamingAssets

Encrypt files in StreamingAssets and load them via UpfEncryptedFileService.LoadStreamingAsset(...).

📦 AssetBundles

Encrypt bundle payloads; UpfEncryptedAssetBundleLoader decrypts and loads them through Unity's memory loading API.

🔗 Addressables

Encrypt local generated bundle files; UpfAddressablesRuntimeBridge decrypts them transparently behind Addressables.

Protecting Addressables content

Configure your Addressables groups, then run protection (from the Dashboard's Addressables button or headlessly):

Unity.exe -batchmode -quit -projectPath . \
  -executeMethod UPF.Editor.Batch.ProtectAddressables \
  -upfProfile Assets/UPFProfiles/DefaultUPFProfile.assetPowerShell

Runtime bridge

UpfAddressablesRuntimeBridge.Install(...) registers an InternalIdTransformFunc. For local protected bundle files it decrypts the UPF payload into a temporary cache file and returns that plaintext cache path to the AssetBundle provider; remote URLs and unsupported schemes pass through unchanged. Install it before your first Addressables load:

using UPF.Runtime;

bridge = UpfAddressablesRuntimeBridge.Install(
    new UpfCryptoService(new StaticUpfKeyProvider(rootMaterial, keyId)),
    context: "addressables");

// Then use Addressables.LoadAssetAsync normally - decryption is transparent.C#
⚠
Remote content needs its own flow. Remote Addressables URLs, Android jar: paths and custom providers are not covered by the local bridge — use UpfRemoteContentService / UpfRemoteContentManifest for signed, hash-verified remote content. Note: AssetBundle and Addressables content can still be captured after it is loaded into memory.

🦷 Runtime Integrity

A signed manifest, verified at startup against a trust anchor shipped in the build.

Protected builds sign the integrity manifest by default. If no automation private key is configured, the build auto-generates an RSA-2048 keypair, signs the manifest, embeds the public key in a shipped upf-runtime-protection.json, and writes the private key only to the report / artifacts folder — never into the player.

Verification at startup

  • UpfRuntimeProtectionGuard.Evaluate(config, rootDataPath) verifies the signed manifest (and optionally runs anti-tamper) and returns a UpfRuntimeProtectionReport with no side effects — fully testable.
  • UpfRuntimeProtectionBootstrap is an opt-in RuntimeInitializeOnLoadMethod that, in a shipped player, loads upf-runtime-protection.json and enforces the configured UpfIntegrityResponse (e.g. fail-closed Application.Quit). It is a no-op in the editor and when no config is shipped.
  • Subscribe to UpfRuntimeProtectionBootstrap.ProtectionEvaluated for custom handling.

Lower-level service

using UPF.Runtime;

var service = new UpfIntegrityService(manifest, root);
UpfResult result = service.VerifyAll();
if (!result.Success) {
    // react per your policy
}C#

UpfBuildFingerprint records product, Unity, build, manifest and profile identifiers; UpfBuildFingerprintVerifier checks the hash of that canonical payload for runtime or support diagnostics.

⚠
Re-sign after every legitimate content update. Editing a file and recomputing its hash no longer passes — but the trust anchor itself ships in the player, so a binary-level attacker can still patch out the check or replace the anchor. This raises the cost of tampering; it does not make it impossible.

🛡 Anti-Tamper

Local tamper / debugging signals you respond to with your own policy.

UpfAntiTamperService emits local signals: managed debugger detection, Android root / Apple jailbreak indicator checks (where platform APIs permit), and memory canaries. You decide the response in your project. Repeated identical signals are throttled (SignalResponseInterval), and DevelopmentBypass is forced off in release player builds.

using UPF.Runtime;

var service = new UpfAntiTamperService();
service.SignalRaised += signal => Debug.LogWarning($"Signal: {signal.Kind} {signal.Message}");
service.RegisterMemoryCanary("session-start");

UpfResult result = service.EvaluateNow();C#

Watched values

RegisterWatchedValue(id, sampler) monitors integrator-supplied bytes — a snapshot of real game state such as currency or health — and raises a ProtectedValueTamper signal when they change in memory. Call RebaselineWatchedValue(id) after a legitimate change.

service.RegisterWatchedValue("gold", () => BitConverter.GetBytes(player.Gold));
// ...after a legitimate purchase:
service.RebaselineWatchedValue("gold");C#
⚠
Not a complete anti-cheat product. These signals do not make runtime memory impossible to inspect or patch, and anti-debugging / environment checks can false-positive. Do not rely on a canary as a sole enforcement mechanism, and never substitute these for server authority. The no-code Protection Guardian wraps this service for zero-code use.

💳 Licensing & Trials

Offline signed-token validation and a rollback-resistant trial window.

Offline license validation

UpfLicenseService validates offline RSA-signed tokens and optional feature entitlements:

using UPF.Runtime;

var token = JsonUtility.FromJson<UpfLicenseToken>(tokenJson);
var service = new UpfLicenseService(productId, publicKeyXml);
var result = service.Validate(token, requiredFeature: "core");
if (result.IsValid) {
    // entitlement granted
}C#

For server-activated licensing, UpfHttpLicenseProvider has a secure constructor overload (endpoint, productId, publicKeyXml, deviceBindingProvider) that verifies the signed token locally and ignores the server's self-reported state, defeating a MITM/proxy that simply returns "valid". Set CertificateHandlerFactory to pin the activation server's certificate. UpfDeviceBinding creates privacy-aware hashed device bindings.

Offline trials

UpfTrialService enforces an offline trial window (the runtime consumer of licensing.trialDays). It records the trial start in authenticated secure storage and measures elapsed time against UpfMonotonicTimeAnchor, a rollback-resistant clock that persists the highest observed time — so moving the device clock backwards cannot extend the trial (it reports ClockTampered).

var trial = new UpfTrialService(storage, trialDays: 14);
UpfTrialStatus status = trial.Evaluate();
// status.IsActive, status.IsExpired, status.RemainingDays, status.ClockTamperedC#
⚠
Trials reset on a reinstall. Clearing app data or reinstalling resets the trial — a documented client-side limitation. Treat trials as a friction / licensing signal, not a hard entitlement boundary, and back real entitlements with a server or platform store.

⚙ IL2CPP

The managed obfuscation pass runs before IL is converted to C++, so protection reaches the native binary.

For IL2CPP standalone builds, the UpfScriptDllPostProcessor (an IPostBuildPlayerScriptDLLs hook, gated by obfuscation.protectIl2CppBeforeConversion) obfuscates the staged user assemblies before il2cpp.exe transpiles them. As a result, renamed private members and constant transforms propagate into the native binary and into global-metadata.dat (verified: protected names are present in the shipped metadata; the original private method / field names are not). UPF's own assemblies (UPF.*) and editor-only assemblies are intentionally excluded.

Validation & metadata shadow

UpfIl2CppValidator reports the active scripting backend and scans build outputs for global-metadata.dat, metadata size / hash, and common native-binary artifacts. When metadata is found it can create an encrypted metadata shadow (global-metadata.dat.upfshadow + a JSON descriptor) for evidence / escrow / tamper-response workflows. It is a validation / fallback workflow — it does not rewrite Unity-generated native C++ or replace global-metadata.dat in place.

# Validate an IL2CPP player build output
Unity.exe -batchmode -quit -projectPath . \
  -executeMethod UPF.Editor.Batch.ValidateIl2Cpp \
  -upfBuildOutput Builds/UPFProtected/Windows/ProtectedGame.exePowerShell
⚠
IL2CPP is not an obfuscator by itself, and some residuals remain. Public type names are kept for Unity serialization; compiler-generated state-machine / iterator types (<Method>d__N) retain the original method name as a substring; and MonoScript source-file paths may remain in metadata. Always run a real IL2CPP player build on each supported platform and validate against the actual output.

🔧 Editor Tools / No-Code

A headline capability: protect your game without writing a single line of code.

✅
Three steps to protect your game. 1) Create a Protection Profile and pick a preset. 2) (Optional) add the Protection Guardian component for runtime anti-tamper. 3) Tools → UPF → Build Protected Player — or use the Dashboard / headless commands.

Welcome window

A first-run onboarding window opens automatically once per project (never in batch / CI) and is re-openable from Window → UPF → Welcome. It links straight to Documentation, the Dashboard, profile creation, dropping a Protection Guardian into the scene, and the demo scene — plus the requirements and support contact.

Protection Profile & one-click presets

The UpfProtectionProfile ScriptableObject is the single source of truth for every layer. Create it via Assets → Create → UPF → Protection Profile or Tools → UPF → Create or Select Protection Profile. Its custom Inspector adds:

  • Quick Setup Presets — one click for Safe, Balanced or Aggressive, each setting a coherent group of obfuscation + strict-mode flags. The full flag catalog stays editable below (nothing is hidden from power users).
  • Key-status callout — warns clearly if asset encryption is still using the default development password, so you can't accidentally ship insecure ciphertext (release / strict builds refuse the default).
  • Shortcut buttons to Open UPF Dashboard and Open Welcome / Help.

Protection Guardian component

The star of the no-code layer. Add it via Add Component → UPF → UPF Protection Guardian or GameObject → UPF → Protection Guardian. Drop it on a GameObject, choose a response, tick the checks you want, and wire UnityEvents — no code.

Inspector fields

FieldWhat it does
checkOnStartRun the configured checks automatically on Start. Turn off to trigger checks yourself (e.g. from a UI Button calling EvaluateNow()).
deferToBuildProtectionIf a protected build already enforces a runtime-protection config at startup, skip re-verifying the manifest here to avoid double enforcement (anti-tamper still runs).
runAntiTamperRun UPF's debugger / environment anti-tamper checks. Works with no extra setup.
repeatIntervalSecondsSeconds between repeated checks while the game runs. 0 = check only once.
verifyIntegrityManifestAdvanced Verify a signed integrity manifest that shipped with a protected build (leave off unless you produced one).
manifestRelativePathFile name (relative to the player data folder) of the signed manifest (default upf-integrity-manifest.json).
publicKeyXmlRSA public key (XML) the manifest signature is verified against — the trust anchor.
requireSignatureReject a manifest that is unsigned or signed with the wrong key.
failureResponseWhat to do when a check fails: ReportOnly, WarnUser, DisableProtectedFeature, CustomCallback or FailClosed.

UnityEvents (no code required)

  • OnVerified — invoked when all configured checks pass. Wire it to enable gameplay, hide a warning, etc.
  • OnTamperDetected (string) — invoked with a failure message when a check fails. Wire it to your warning panel, a save-disable, or any custom reaction.

You can also call EvaluateNow() from a Button's OnClick, and read LastResultPassed / LastMessage.

var key = UpfCryptoKey.FromPassword("sample", password,

        UpfBytes.Combine(key.EncryptionKey, key.AuthenticationKey), "sample"));

var sealed_ = crypto.Protect(plaintext, "sample", "sample-save");

⚠ Security Model & Limits

Read this before you ship. UPF is honest about what it can and cannot do.

UPF is a defense-in-depth toolkit. Client-side protections raise the cost of casual inspection and tampering, but they do not make a client build immune to reverse engineering. A determined attacker can inspect memory after decryption, patch local checks, dump runtime data, or analyze native binaries.

What UPF does not do

  • Client-side keys can be extracted by skilled attackers, and runtime-decrypted data can be captured from memory.
  • The signed integrity manifest's trust anchor ships in the player, so a binary-level attacker can still patch out the check or replace the anchor.
  • Offline trials reset when app data is cleared or the app is reinstalled (clock rollback is resisted, reinstall is not).
  • Constant encryption currently covers direct ldc.i4, ldc.i4.s, ldc.r4 and ldc.r8 instructions — not array initializers, serialized fields, shader data, or native constants.
  • Control-flow noise is limited to branch / dead-code insertion in methods without exception handlers — not commercial control-flow flattening.
  • Anti-debugging and environment checks can false-positive; WebGL secure-storage and anti-tamper options are limited.
  • AssetBundle and Addressables content can be captured after runtime load. IL2CPP is not an obfuscator by itself.
  • UPF does not replace server-authoritative multiplayer security.
✅
Use it alongside the right architecture. Pair UPF with server-authoritative multiplayer, secure backend validation, platform-store licensing, CI-managed signing keys, and clear key-rotation processes. Provision asset-encryption and signing keys from outside the repository (CI environment variable or external key file) — the default development password is rejected by release / strict readiness validation.

💡 Troubleshooting

The issues that come up most, and how to fix them.

Protected build fails / breaks after renaming

Run Analyze in the Dashboard and review the reflection, UnityEvent, animation-event and serialization findings. Add obfuscation.preserveNamePatterns entries for string-based references and serialized DTOs, or start from the Safe preset and widen gradually. A Dry Run shows exactly what would change before you build.

A secure save can't be loaded

Confirm the key id, context and associated data match what was used to write the save. Tampered or mismatched saves fail authentication by design. If you renamed a protected asset used as associated data, regenerate it.

Integrity / manifest validation fails after a content patch

Regenerate and re-sign the integrity manifest after every legitimate content update. A changed file with a recomputed hash will fail until the signed manifest is rebuilt.

A batchmode command doesn't run right after import

Unity performs a script-compilation pass first after an .asmdef or package change. Wait for compilation to finish, then run the command again.

Addressables protection is skipped

Install the Addressables package and configure protected groups first. UPF does not require Addressables for the base package; protection runs only when Addressables is present and configured.

An obfuscated stack trace is unreadable

Decode it with UPF.Editor.Batch.DecodeStackTrace using the UPF_Mapping.csv produced by the protected build. Keep that mapping file private and out of distributed packages.

❓ FAQ

Quick answers to the questions that come up most.

Do I need to write code to use UPF?

No. Create a profile, pick a preset, and build from Tools → UPF → Build Protected Player or the Dashboard. For runtime protection, drop the UPF Protection Guardian component on a GameObject and wire its OnVerified / OnTamperDetected UnityEvents — entirely no-code. The scripting API is there when you want it.

Does UPF guarantee my game can't be cracked?

No, and we won't claim otherwise. UPF is defense-in-depth: it raises the cost of casual inspection and tampering. Client-side keys can be extracted and decrypted data can be read from memory. See Security Model & Limits.

Which Unity versions and backends are supported?

Unity 6000.0 (Unity 6) or newer, developed and validated against 6000.0.62f1. Both Mono and IL2CPP standalone builds receive the managed-assembly obfuscation pass.

Does obfuscation break Unity serialization or reflection?

The pass is conservative around Unity serialization, reflection, Unity messages, compiler-generated code and methods with exception handlers. Run Analyze first and add preserve rules for anything the scanner flags (string-based reflection, serialized DTOs, public API names).

Can I run UPF in CI without opening the editor UI?

Yes. Every workflow is available as a UPF.Editor.Batch.* command (Analyze, DryRun, ProtectBuild, ProtectAddressables, ValidatePackage, FinalAudit and more). The Dashboard shows copy-ready command strings, and a player self-test (-upfSelfTest) writes a RESULT=PASS/FAIL file for pipelines.

How do I provision a production encryption key?

Set the environment variable named by encryption.productionKeyEnvVar (default UPF_ASSET_KEY) in CI, or point encryption.productionKeyFilePath at an external key file. Release / strict builds refuse the default development password. Keep all keys and the mapping CSV out of version control.

Is there a demo I can play?

Yes — open Assets/UPF/Demo/Scenes/UPF_InteractiveDemo.unity (or Tools → UPF → Open Demo Scene) for a runtime feature showcase, plus the Assets/UPF/Samples scripts for secure saves, licensing, integrity, Addressables and anti-tamper.
