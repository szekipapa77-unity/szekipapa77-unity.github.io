UpscaleOS

The Universal Upscaling Layer for Unity — one component/API for the major upscalers with automatic GPU detection and an open-source core that runs on every GPU with zero native DLLs.

Unity 6000.0+ Open core · Zero native DLLs URP / HDRP / Built-in DLSS · FSR · XeSS · SGSR
ⓘ
One layer, every GPU. Add one component (or call one API) and UpscaleOS detects the player's GPU and picks the best available upscaler. The open-source core — FSR 1, a vendor-agnostic temporal reconstructor, NIS-style, SGSR 1/2 and a Bilinear floor — runs on every GPU with no native DLLs. The vendor ML paths (DLSS / FSR 4 / XeSS) are optional.
✅
It can never hard-fail. An automatic fallback chain always lands on a backend that runs — ending at Bilinear — so there is no GPU where UpscaleOS fails to render. See Backends & Automatic Fallback.

⤓ Installation

Import the package, run the Setup Wizard, add the component.

Import from the Asset Store
In Window → Package Manager → My Assets, download and import UpscaleOS. It lands at Assets/UpscaleOS/.
Open the Setup Wizard
Tools → UpscaleOS → Setup Wizard runs colour-coded project diagnostics with one-click fixes. On URP, click Add UpscaleOS to all URP renderers.
Add the renderer path
The wizard installs the UpscaleOSRendererFeature on your URP renderer(s); the feature requests the depth and motion vectors that temporal upscalers need.
Add the component & play
Add the Upscale Controller to your camera (or use GameObject → UpscaleOS → Upscaling Camera), pick a Quality preset, and press Play.

Requirements

  • Unity 6000.0 or newer.
  • URP 17 for the one-click path, or HDRP 17 via a CustomPass. The Built-in pipeline has a driver (OnRenderImage based).
  • The open core needs no native DLLs and runs on every GPU.
  • Vendor ML backends (DLSS / FSR 4 / XeSS) are Windows x64 + the vendor SDK + the native bridge you build — entirely optional.

⚡ Quick Start (URP)

The primary one-click path: a renderer feature plus one component.

Add UpscaleOS to your renderers
Tools → UpscaleOS → Setup Wizard → Add UpscaleOS to all URP renderers. This adds the renderer feature (which requests depth + motion vectors) to every URP renderer.
Add the Upscale Controller
On your main camera: Add Component → UpscaleOS → Upscale Controller. (Or drop a ready-made camera with GameObject → UpscaleOS → Upscaling Camera, which also installs the feature.)
Pick a Quality preset
Choose a Quality mode (e.g. Balanced) — or enable Smart Mode to let UpscaleOS benchmark and choose for you.
Press Play
UpscaleOS detects your GPU and selects the best available backend automatically. Use Tools → UpscaleOS → Diagnostics to confirm what was chosen.
💡
Validate motion vectors before you ship. Temporal upscalers depend on correct motion vectors. Run Tools → UpscaleOS → Validate Motion Vectors to catch ghosting risks early.

⚙ How It Works

Render the scene below display resolution, reconstruct it, present at display resolution.

Render low-res → + Jitter → + Motion vectors → Reconstruct → Sharpen (RCAS) → Display-res
  1. The pipeline renders the scene at a reduced internal render scale (e.g. 67% for Quality).
  2. A sub-pixel jitter is applied each frame so successive frames sample different positions.
  3. The selected backend reconstructs the full display resolution image. Temporal backends additionally consume motion vectors (and depth) to reuse detail from previous frames; spatial backends upscale the single frame.
  4. An optional RCAS sharpening pass cleans up the reconstructed result.
  5. The pass injects before post-processing, so UI drawn afterwards stays crisp at native resolution by default.
ⓘ
The temporal path is what removes aliasing and recovers detail. That is why jitter + motion vectors feed it — without valid motion vectors, temporal backends can ghost or smear (see Troubleshooting).

💾 The Quality Preset Asset Workflow

A reusable ScriptableObject for standardising settings across scenes and projects — a team-friendly workflow.

Create the asset
Assets → Create → UpscaleOS → Quality Preset makes a Quality Preset ScriptableObject in your project.
Assign it to a controller
Drag it into the controller's Quality Preset slot. When assigned, it is applied automatically on enable.
Reuse it everywhere
Share the same preset across scenes/projects so every camera uses identical upscaling settings.

Inspector buttons

The Upscale Controller inspector has helper buttons so you never have to hand-edit the asset:

  • Apply preset to fields — push the assigned preset's values onto the controller's fields.
  • Capture fields into preset — write the controller's current fields back into the preset.
  • Create & assign preset — make a new preset from the current fields and assign it in one click.
💡
Code-first still wins. An assigned preset applies on enable, but anything you set in Start() / at runtime overrides it — presets and code coexist.

Each preset maps to an upscale factor and an internal render percentage — lower render % means more performance.

ⓘ

🔌 Backends & Automatic Fallback

An open-source core that runs everywhere, optional vendor ML paths, and a fallback chain that can never hard-fail.

Automatic fallback chain

UpscaleOS picks the best available backend for the GPU and always falls back so it can never hard-fail to render:

NVIDIA RTX  → DLSS    // only if the vendor DLL + bridge are present
AMD RDNA4   → FSR4    // only if the vendor DLL + bridge are present
Intel Arc   → XeSS    // only if the vendor DLL + bridge are present

desktop, no vendor DLL  → UpscaleOS-TU
mobile                  → SGSR2 → SGSR1
weakest / unknown       → FSR1 → BilinearFallback
✅
No closed binaries ship. Activating a vendor path is entirely your choice: build the included bridge source, add the vendor SDK + DLLs under that vendor's license, and the backend activates automatically. With nothing added, the open core runs — everywhere.

🧠 Smart Mode

Let UpscaleOS measure your GPU and choose for you.

When Smart Mode is enabled, UpscaleOS benchmarks every backend available on the player's GPU at startup and selects the highest-quality one whose cost fits your frame budget. It overrides Preferred Backend — so you don't have to guess which upscaler is best for a given machine.

ⓘ
Smart Mode runs once at startup. To re-run the selection from code, call RunSmartSelection() on the controller.

📈 Adaptive Resolution

Hold a target frame rate by continuously nudging the render scale.

When Auto-Tune To Target FPS is on, a governor continuously nudges the internal render scale to hold your Target FPS. It is:

  • CPU/GPU-bound aware — it won't lower the render scale when you're CPU-bound (lowering it wouldn't help).
  • Anti-oscillation guarded — it avoids hunting back and forth around the target.
var c = camera.GetComponent<UpscaleController>();
c.autoTuneToTargetFps = true;
c.targetFps          = 60;   // 30..240
c.ApplySettings();C#

✨ Sharpening

RCAS after reconstruction, plus an optional extra post-sharpen pass.

  • Sharpness — an RCAS sharpness amount applied after reconstruction (0–1).
  • Post-Sharpen — an optional extra CAS/RCAS pass, with its own strength, to sharpen temporal or vendor results that you want a bit crisper.
ⓘ
Both controls live on the Upscale Controller and can be driven from code via the matching fields plus ApplySettings().

🎥 Frame Generation

UpscaleOS ships the motion-vector interpolation and frame-pacing logic — honestly reported.

UpscaleOS includes the motion-vector interpolation + even frame-pacing logic and an IFrameGenPresenter hook. However, actually presenting synthesized frames needs a platform/swapchain present implementation, so on the open path the toggle does not add frames by itself — it reports "requested" honestly.

⚠
Vendor frame generation (DLSS-G / FSR-FG) requires the optional native bridge. Note that DLSS-G frame generation requires an RTX 40-series GPU.

🌐 Multi-Pipeline (URP / HDRP / Built-in)

One asset, three render pipelines on Unity 6000.0+.

URP 17 Primary

The primary one-click path. UpscaleOS plugs in as a ScriptableRendererFeature — the Setup Wizard's Add UpscaleOS to all URP renderers adds it for you, and it requests the depth + motion vectors temporal upscalers need.

HDRP 17 CustomPass

HDRP has no renderer features, so UpscaleOS plugs in as a CustomPass:

Add a Custom Pass Volume
GameObject → Volume → Custom Pass, set its injection point to Before Post Process, and add the Upscale HDRP Pass to it.
Pick Quality (and optionally a backend)
Set a Quality preset and an optional preferred backend on the pass.
Pair with HDRP Dynamic Resolution
For sub-native rendering, enable Dynamic Resolution on the HDRP Asset and the camera; the pass reconstructs the DRS-scaled colour up to display resolution.

Built-in Driver

The Built-in pipeline has a driver based on OnRenderImage.

💡
UI stays crisp. The upscale pass injects before post-processing, so UI drawn afterwards is presented at native resolution by default.

📙 Scripting API — UpscaleService

The static entry point. Namespace: UpscaleOS. Code-first usage is fully supported.

Common usage

using UpscaleOS;

// Query the environment
Debug.Log(UpscaleService.Gpu);       // detected vendor / tier / device
Debug.Log(UpscaleService.Active);    // selected backend
foreach (var b in UpscaleService.Available) Debug.Log(b);

// Configure quality + display resolution
UpscaleService.Configure(QualityMode.Quality, new Vector2Int(1920, 1080));C#

Members

MemberDescription
UpscaleService.GpuThe detected GPU (vendor / tier / device).
UpscaleService.ActiveThe currently selected backend.
UpscaleService.AvailableIterable set of backends available on this GPU (foreach).
UpscaleService.Configure(QualityMode, Vector2Int)Set the quality mode and display resolution.
UpscaleService.ExplainSelection()A per-backend why-chosen / why-rejected trace.
UpscaleService.ResetHistory()Flush temporal history — call on a hard cut / teleport.
UpscaleService.ActiveRenderStatsLive render-side statistics.

// On a camera cut / teleport, drop stale temporal history:

💡 Troubleshooting

The issues that come up most, and how to fix them.

Ghosting / smearing behind moving objects

Temporal upscalers (UpscaleOS-TU, DLSS, FSR 4, XeSS) need correct motion vectors. Run Tools → UpscaleOS → Validate Motion Vectors to find problems. Custom and vegetation shaders need a MotionVectors pass — without one, animated geometry has no motion data and ghosts.

No upscaling is happening at all

On URP, the renderer feature isn't added. Open Tools → UpscaleOS → Setup Wizard and click Add UpscaleOS to all URP renderers.

A vendor backend (DLSS / FSR 4 / XeSS) isn't active

Its vendor DLLs / native bridge aren't present. The Diagnostics window shows what's installed. UpscaleOS falls back to the open core — this is expected until you build the bridge and add the vendor SDK + DLLs.

HDRP shows no upscaling

Add a Custom Pass Volume with the Upscale HDRP Pass (injection point Before Post Process), and enable Dynamic Resolution on the HDRP asset + camera. See Multi-Pipeline.

The UI looks soft

The upscale pass injects before post-processing so UI stays crisp by default. If your UI is soft, confirm it is drawn after the upscale pass (e.g. a Screen Space – Overlay canvas, or post-upscale) rather than inside the upscaled scene buffer.

❓ FAQ

Quick answers to the questions that come up most.

Do I need vendor DLLs?

No. The open core works on every GPU with zero DLLs. The vendor ML paths (DLSS / FSR 4 / XeSS) are entirely optional — you only build the native bridge and add the vendor SDK if you want those specific backends.

Does it work without coding?

Yes. Add the Upscale Controller component (or use GameObject → UpscaleOS → Upscaling Camera), pick a Quality preset, and press Play. No scripting required.

Which Unity version do I need?

Unity 6000.0 or newer.

Which render pipelines are supported?

URP (the one-click path via a renderer feature), HDRP (via a CustomPass), and Built-in (via a driver).

Does it support consoles / XR?

The open core is portable; vendor and console paths depend on the relevant platform SDKs.

Does Frame Generation actually add frames?

On the open path, the toggle reports "requested" but does not add frames by itself — presenting synthesized frames needs a platform present hook (IFrameGenPresenter). Vendor frame generation (DLSS-G / FSR-FG) requires the optional native bridge, and DLSS-G needs an RTX 40-series GPU.
